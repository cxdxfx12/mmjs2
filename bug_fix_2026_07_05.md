## 1. 活动规则过期检查遗漏 [CRITICAL]

**文件**: `internal/rules/repository.go:393-401`

**问题描述**:
`FindBestRule` 函数中，`isCampaignActive()` 检查只在第二轮（通配省份匹配）执行，第一轮（精确省份匹配）缺失。导致已过期的活动规则在精确匹配省份时仍会被选中。

**修复代码**:
```go
// 第一轮：精确省份匹配
for _, r := range enabled {
    if r.RuleType == ruleType {
        if ruleType != "global" && NormalizeCustomerName(r.CustomerName) != custKey {
            continue
        }
        // 新增：活动规则过期检查
        if ruleType == "campaign" && !isCampaignActive(r) {
            continue
        }
        if r.Province != "" && NormalizeProvince(r.Province) == provKey {
            return &RuleResult{Rule: r, RuleLevel: ruleType}
        }
    }
}
```

**影响**: 确保过期活动规则不会被错误应用

## 2. 规则更新时无法重置字段为 0 [HIGH]

**文件**:
- `internal/rules/repository.go:180-188`
- `internal/app/app.go:171-217`

**问题描述**:
`MinFee`/`MaxFee`/`Surcharge` 字段无法重置为 0，因为原代码用 0 值判断是否保留原值。

**修复方案**:
1. `repository.go`: 使用 -1 作为"未设置"标记
```go
// 使用 -1 作为"未设置"标记，允许将字段重置为 0
if r.MinFee < 0 {
    r.MinFee = existing.MinFee
}
if r.MaxFee < 0 {
    r.MaxFee = existing.MaxFee
}
if r.Surcharge < 0 {
    r.Surcharge = existing.Surcharge
}
```

2. `app.go`: 改为指针类型区分"未设置"和"0"
```go
type RuleSaveReq struct {
    // ... 其他字段 ...
    MinFee    *float64 `json:"min_fee,omitempty"`
    MaxFee    *float64 `json:"max_fee,omitempty"`
    Surcharge *float64 `json:"surcharge,omitempty"`
    // ...
}
```

**影响**: 用户现在可以将保底价、最高价、附加费正确重置为 0

## 5. 批量删除错误被忽略 [MEDIUM]

**文件**: `internal/rules/repository.go:262-268`

**问题描述**:
`DeleteBatch` 循环调用 `Delete` 但忽略所有错误，调用方无法获知删除失败。

**修复代码**:
```go
func DeleteBatch(ids []int64) error {
    var lastErr error
    for _, id := range ids {
        if err := Delete(id); err != nil {
            lastErr = err
        }
    }
    return lastErr  // 返回最后一个错误
}
```

**影响**: 批量删除操作现在可以正确报告错误
