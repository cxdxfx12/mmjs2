## 1. 规则管理页（`frontend/src/views/Rules.vue`）

### 1.1 页面结构（未改架构）

- 路由：`/#/rules`
- 布局：上（全局规则，可折叠）+ 下（左客户列表 / 右规则编辑）
- 右栏两种视图：**阶段报价**（`viewMode='zone'`）、**列表视图**（`viewMode='list'`）

### 1.2 右栏高度与 34 省滚动

**目标**：尽量在一屏内浏览全部 34 个省份规则。

**实现**：

| 项 | 说明 |
|---|---|
| 页面高度 | `.rules-page { height: calc(100vh - 142px) }` |
| 表格高度 | `:max-height="ruleTableMaxHeight"`，`ruleTableMaxHeight = 'calc(100vh - 280px)'` |
| 紧凑行高 | `.rules-compact-table` 行高约 22px |
| 拉均重 | 从表格列移到 **右栏工具栏** 单行显示，减少行高 |

**彩虹滚动条（重要踩坑）**：

- Element Plus 2.x 表格内部使用 **`el-scrollbar` 自定义滚动条**，不是浏览器原生 `::-webkit-scrollbar`。
- 仅给 `.el-table__body-wrapper::-webkit-scrollbar` 写样式 **无效**。
- 正确做法（见 `Rules.vue` 底部**非 scoped** `<style>` 块）：
  - 样式 `.el-scrollbar__bar` / `.el-scrollbar__thumb`（彩虹渐变）
  - 同时保留 `.el-scrollbar__wrap::-webkit-scrollbar` 作为 `native-scrollbar` 备用
- 表格需加：`scrollbar-always-on`、`native-scrollbar`

### 1.3 「启用 / 操作」列显示/隐藏

**需求演变**：先尝试鼠标移入自动显示 → 在 exe 内置浏览器中**不可靠**，最终改为开关控制。

**最终方案**：

```vue
<!-- 工具栏 -->
<span class="rh-action-toggle">
  <span class="rh-action-label">操作列</span>
  <el-switch v-model="showActionCols" size="small" />
</span>

<!-- 表格列 -->
<el-table-column v-if="showActionCols" label="启用" ... />
<el-table-column v-if="showActionCols" label="操作" ... />
```

| 项 | 说明 |
|---|---|
| 默认值 | `showActionCols = ref(true)`，进入页面默认显示 |
| 强制刷新 | `:key="\`zone-${showActionCols}\`"` / `:key="\`list-${showActionCols}\`"`，避免 EP 表格动态列不更新 |
| 布局刷新 | `watch(showActionCols)` → `rulesTableRef.doLayout()` |

**失败方案（勿再用）**：

- CSS `display:none` 隐藏 `class-name` 列 → EP 表格单元格类名不稳定，常无效
- `fixed="right"` + `transform: translateX` 隐藏 → 被 EP 布局 JS 覆盖
- 仅 `@mouseenter` / `@mouseleave` 控制 → 嵌入式 WebView 事件不可靠

## 3. Windows 打包

### 3.1 输出文件名

- 产物：**`喵喵云结算.exe`**（项目根目录）
- 脚本：`build.ps1`、`build.bat`

**中文文件名编译踩坑**：

```powershell
# 错误：go build -o "喵喵云结算.exe"  → PowerShell/Go 编码乱码
# 正确：先编译英文临时文件再重命名
go build -o _build_temp.exe .
[System.IO.File]::Move($tempExe, $exePath)
```

PowerShell 脚本内 exe 名用 Unicode 码点构造，避免脚本文件编码问题：

```powershell
$exeName = (-join [char[]]@(0x55B5, 0x55B5, 0x4E91, 0x7ED3, 0x7B97)) + ".exe"
```

### 3.2 构建命令

```powershell
# 项目根目录
.\build.ps1
```

流程：生成图标 syso → `npm run build`（frontend）→ `go build -ldflags="-s -w -H windowsgui"`

### 3.3 EXE 图标（最终有效方案）

**现象**：用 `ico_to_syso.py` 生成的 `rsrc_windows_amd64.syso` 编译后 exe **无图标**。

**有效方案**：使用 Go 生态标准工具 **`rsrc`**：

```powershell
go install github.com/akavel/rsrc@latest
rsrc -ico monkey.ico -arch amd64 -o rsrc_windows_amd64.syso
go build -ldflags="-s -w -H windowsgui" -o _build_temp.exe .
```

`build.ps1` 已改为**优先调用 rsrc**，`ico_to_syso.py` 仅作备用。

**要点**：

| 项 | 说明 |
|---|---|
| 图标源 | `monkey.ico`（项目根） |
| 资源文件 | `rsrc_windows_amd64.syso`（须此命名，Go 自动链接） |
| 图标缓存 | Windows 资源管理器可能缓存旧图标 → 关闭旧 exe、F5 刷新文件夹 |

## 6. 后续改动建议

1. **操作列**：若需「鼠标移入显示」，建议与「操作列」开关并存，不要单独依赖 hover。
2. **图标**：新环境首次打包需 `go install github.com/akavel/rsrc@latest`。
3. **滚动条样式**：升级 Element Plus 后检查 `el-scrollbar` DOM 类名是否变化。
4. **参考价**：若需可配置参考价，可新增 SQLite 表或 JSON 配置，替代纯硬编码 `GetSamplePriceTable()`。
